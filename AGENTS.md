# AGENTS.md — operating manual for AI agents in this repo

Multi-mod monorepo for **Dawncaster: The RPG Cardventure** (Unity Mono, BepInEx 5 +
HarmonyX). One engine package (`DC.DawnKit`: DLLs + data contracts), data-only content
packs (`DC.*` dirs with a `pack.json`), shared Python toolchain in `tools/`,
decompile-verified knowledge in `docs/research/`. Human-facing overview: `README.md`;
direction/vision: `MOD-TOOLKIT.md`.

## Binding rules (never violate)

1. **The game directory is READ-ONLY** (`E:\Games\Steam\steamapps\common\Dawncaster`).
2. **Canonical typos are API surface** — `CardRariry`, `Phyisical`,
   `conditonOperator`, etc. Never "fix" spellings in data or when reading the game's
   enums. (The public C# API exposes corrected spellings and maps internally.)
3. **Closed vocabularies:** every DSL command must exist in
   `docs/research/reference/` (effect/talent/dialogue command lists). No invented
   vocabulary, ever.
4. **IDs only from allocated blocks** in `docs/ID-REGISTRY.md` (mods own
   700,000,000–799,999,999; cards bottom-up, weapons/powers/starting cards share one
   top-down cursor per block).
5. **Claims about the shipped pool require scripted scans** over `tools/out/data/`,
   not memory. Claims about mechanics require decompile/asset citations.
6. **Gates are the definition of done** for content:
   `dmk validate` → `dmk artcheck --distinctness` → `dmk sim` → in-game QA. Nothing
   ships on a red gate; `python -m pytest tools/tests` must pass before toolchain
   changes land.
7. **Art PNGs and `tools/out/`/`decompiled/` are gitignored, rebuildable artifacts** —
   only recipes/manifests travel. Never commit extracted or derived game assets.
8. **Fail-safe engine rule:** every engine feature has an `Enabled` knob; parse
   failures skip the item, log loudly, and leave vanilla behavior untouched.

## Commands

```
python tools/dmk.py -h            # single entry point for the whole chain
python tools/dmk.py new "<Name>" --author "<you>"   # scaffold a pack
python tools/dmk.py validate --all                  # gate 1: schema/enums/collisions
python tools/dmk.py artcheck --all --distinctness   # gate 2: art shipping + phash
python tools/dmk.py sim --all                       # gate 3: balance envelopes
python tools/dmk.py all <pack>                      # full gate chain for one pack
python -m pytest tools/tests                        # toolchain test suite
```

Game dir: `E:\Games\Steam\steamapps\common\Dawncaster` (read-only). Runtime feedback:
`BepInEx\LogOutput.log`. Regenerate `decompiled/` and `tools/out/` after game updates
(`tools/extract_data.py`, `tools/extract_sprites.py`).

## Output artifacts by task

- Content packs ship `pack.json` + `DESIGN-NOTES.md` (evidence per METHODOLOGY's
  checklist) + `BALANCE-REPORT.md` (generated by gate 3) + `art-recipes.json`.
- Engine specs follow the SPEC template in `docs/CONVENTIONS.md`.
- Corpus-derived numbers belong in `docs/design/CORPUS-STATS.md` — never inside a
  single pack's notes.
